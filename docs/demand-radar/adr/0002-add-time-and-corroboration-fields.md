# 0002 — 新增「发布时间」「佐证链接」两字段（13 → 15 字段）

- 状态：Accepted
- 日期：2026-07-08
- 相关：`0001-simplified-schema.md`、
  `plugins/jkk/skills/demand-radar/references/feishu-schema.md`、
  `plugins/jkk/skills/demand-radar/scripts/run.py`

## 背景

2026-07-08 对 demand-radar 做数据源扩充与筛选逻辑升级时，发现精简 schema 缺两块信息：

1. **无时间维度**：表里完全没有证据的发布时间，无法判断需求新鲜度，也无法执行
   新加入 evidence gate 的时效规则（>18 个月需近期复现）。
2. **普遍性无凭据**：`需求普遍性=高` 是纯判断，没有跨源复现的「小票」可查证。

## 决策

新增两字段，遵守 ADR-0001 的反空字段原则：

- **`发布时间`**（date，毫秒时间戳，必填）：证据原文的发布时间。帖子/评论/评价
  都自带时间戳，采集时总能拿到，不会成为空字段。`run.py` 校验为
  [1e12, 5e12) 的整数（排除秒级时间戳）。
- **`佐证链接`**（text，可空）：同一 job 跨源复现的第二/第三条 URL。仅在有复现时
  填；空 = 单源，本身就是信息，不算空字段堆积。填了则每个空白分隔的 token 必须是
  http(s) URL（`run.py` 校验）。

明确**不加**：目标用户 / 现有替代方案 / JTBD 单列字段——已内含于
一句话痛点 + 原文内容，单列必然重复表达（正是 ADR-0001 砍掉的原因）。

## 后果

- ✅ 时效规则可执行；需求普遍性有凭据可回查。
- ✅ 校验仍集中在 `run.py`，self-check 覆盖新字段的正反用例。
- ⚠️ 飞书 Base 需实际补建这两个字段（`+field-create`）；此前已写入的存量行这两列
  为空，属迁移遗留，不视为数据质量问题。
