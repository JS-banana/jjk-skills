## 调研发现摘要

### 核心原则：少即是多

- 自动生成的 AGENTS.md 降低任务成功率 ~3%，增加成本 20%+（ETH Zurich 2025）
- 人工编写的 AGENTS.md 仅提升 ~4%，仍增加成本 19%
- LLM 可靠遵循约 150-200 条指令，agent harness 已占用约 50 条
- 推荐：总文件 < 300 行（HumanLayer 的只有 60 行）

### 应该写什么（六核心领域）

1. 项目概述（WHAT）- 技术栈及版本
2. 关键命令（HOW）- 放最前面
3. 非显而易见的模式（WHY）- 信号密度最高
4. 编码规范（HOW）- 只写项目特有约定
5. 边界规则（WHAT NOT）- Always/Ask First/Never
6. 测试规则（HOW）- 精确命令

### 不应该写什么

- 代码风格指南（用 linter 替代）
- 详细的目录列表（agent 可自己发现）
- 通用最佳实践（"写干净的代码"毫无意义）
- 完整的 API 文档（会过时，浪费 token）
- 任务特定的临时指令（放在 prompt 中）
- 自动生成的内容（弊大于利）

### 渐进式披露

- 根目录 AGENTS.md 只放全局通用规则
- 子目录放模块特定规则
- 独立文件放任务特定文档
- 工具自动加载最近目录的 context 文件

### AGENTS.md vs CLAUDE.md

- AGENTS.md：跨工具通用格式（60+ 工具支持）
- CLAUDE.md：Claude Code 专用
- 最佳实践：以 AGENTS.md 为单一事实来源，CLAUDE.md 通过符号链接桥接
