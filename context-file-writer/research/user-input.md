# 用户原始输入

## 用户需求描述

按照类似的思路，你帮我研究分析下怎么写好项目的 AGENTS.md/CLAUDE.md 这个上下文文件的内容，我经常写项目的时候都会遇到这种情况，你帮我仔细研究下，先全网收集各种资料，包括写法规范、原则、内容格式、最佳实践等等，越全面越好；

比较常见的几个场景有：
1. 项目新创建时，怎么根据我的需求想法，编写这个文件，分清主次，任务顺序等，完善这个 agent 上下文文档，也就是上面说的那俩文件，主要是提供给 claude、codex 这类 agent 工具使用的
2. 项目推进的过程中，怎么保证根据项目的进展情况和已经完成功能、待办和未来规划等，对这个文件进行合理的更新
3. 后续项目完成或者阶段性完成，怎么根据想法和功能，继续编写和更新这个文档呢

可能还有各种其他方向，你不能单单根据我想到的这些，还有我没想到的，你要根据我们这次任务情况，补充和拓展哪些需要考虑的、哪些是必要的，然后规划到研究中一起分析研究

上述这些内容，你应该先分析我的描述和想法，然后补充和拓展，基于我这些内容和问题深入了解，然后深度思考，我建议你开始行动前，在综合这些数据后，至少思考 5 轮以上，然后再给出更充分的方案，再开始考虑如何研究，研究哪些内容和方向，怎么更好的使用这些工具和 agent
